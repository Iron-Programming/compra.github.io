What is programming?

Introduction - Computers and Programs

Welcome to programming with the C++ language! Before we go on to study this specific language, C++, we need to get a little bit of a computer background, so let's get started!

A computer is a device that makes computations--that is, it performs mathematical calculations. An defining characteristic of a computer is that it is programmable.

When we give instructions to a computer, we are programming it. Modern computers are assigned sets of instructions, called programs, which they execute. When given programs to execute, computers can execute them automatically without human assistance. This is what makes computers so powerful: we can tell our computers to do something, and they will always do what we say!

At the same time, translating our ideas into a program requires knowledge of how to convert our ideas into a programming language. Many times, we know exactly what we want the computer to do, but we struggle on how to tell it so. If you ask any programmer, they will tell you that debugging, the process of resolving errors, takes up the majority of the time of programming. We will take a look at debugging and troubleshooting later.

There is a fun little poem that describes this situation.

Of course, we don't need to be saying hate, but the quote is pretty humorous.

Writing Programs

To write a program, we must go be able to convert an idea in our head to concrete code. Before we can do that, we must better understand what it is that we want our code to do. There are a couple programming models that programmers use. The two most common are the waterfall model and the spiral model. We will look at programming from the perspective of the waterfall method.

In the waterfall method, there are two general phases. The first phase is the planning or problem-solving phase, and the second phase is implementating that refined idea into a concrete solution (program).

Problem-Solving Phase (Planning Phase)

The problem-solving phase can be understood as a loop. We continue our planning for each aspect of our project until we have enough planned to continue on to implementing our solution.

  • Analyze and Specify → The first step of problem solving is to analyze the situation, so that we can define the problem, and specify what we must do to accomplish our goal.
  • Create a General Algorithm → After we understand what we seek to accomplish, we must develop a general solution (algorithm) to meet this goal. Many times, we will use pseudo-code to help use think more in terms of programming, without getting language specific.
  • Verify → Before we convert our algorithm into code, we should verify our logic to make sure we don't have any flaws. Always expect the unexpected!
Implementation Phase

Once we have verified what our algorithm is, we must convert this "idea" into code, creating a program. This can be thought of in two steps: (1.) translating the algorithm into a programming language, and (2.) testing the program.

After we initially develop a program, a burden may seem to lift off your back. But... you aren't done yet. Depending on the purpose of the program, it may require a little or a lot of maintenance. Maintenance can be considered a third phase of programming, and can be really important in some situations. In some cases, it can be as simple as fixing glitches in a game that your viewers find. In other cases, you may be updating software to stay up to date with operating systems, or maybe changing it to make it compatible with new apps.
Later we will look a little more at algorithms and how we use them in various ways.

Programming Languages

We just described the basic process of how programmers go about coding. But as we noted, we must use a programming language to instruct the computer, because the computer cannot understand human, real-life languages. While there are various forms of computer languages, for now we will only focus on programming languages.

  • Programming Language → A group of rules and characters that are used to construct a computer program.

We describe programming languages by talking about their syntax and semantics. We will look more into this later.

Levels of Programming Languages

Now, there are various levels of programming. Originally, the only programming languages that existed were the ones built into the computers, called machine languages. These languages were the primitive instruction sets built directly into the computers. As we learn in Computer Science and other IT related courses, computers operate on binary. So machine language was written in binary. This made programming very cumbersome and prone to error.
But over time, programmers developed a system that was a lot easier to work in. They developed assembly languages. These languages could not be directly by the computer, so a special program called an assembler was used to run it. Assembly languages used small words or mnemonic phrases to help make programming slightly more human friendly. Now an assembly language is still considered a very low-level programming language. We call programming languages that are "closer" to how the machine operates (on binary) low-level programming languages, and more human-friendly languages high-level.

So because assembly languages are still not very useful friendly, programmers developed high-level languages like C. These languages use a compiler to convert the high-level code directly into machine code. We call a program written in a high-level language a source program, and the program that is actually executed by the computer a object programThis means that the compiler translates the source program into the object program, which is then executed by the computer.

The benefits of using a high level language include:

  • Readability - more human friendly, and less prone to errors.
  • Portability- we can write machine-independent code.
  • Compatibility - it is easier to write programs that are compatible with other programs/software.

Before we leave this section, it is probably worth noting the different types of instructions that we write in programming languages, because we will reference this later.

  • Take input data from an input device (keyboard, mouse, etc) and output data to an output device (monitor, headphones, etc).
  • Store and retrieve data into/from computer memory (primary or secondary).
  • Transfer data from one place to another.
  • Compare two data values (equality/inequality).
  • Perform arithmetic operations (we'll look at these in more depth later).

As we continue studying programming languages in general, and C++ specifically, we will see how all programming statements fall into these categories.

Syntax and Semantics.

Both programming and real-life languages follow a set of rules which dictate how we use them. When discussing how languages work, we typically talk about their syntax, which is the actual structure or grammar of the languages, and semantics, which is the meaning of a "word or phrase."
Now, as we study languages, or programming languages in particular, we will special language that we use to talk about languages. To help us make specifications about a particular language's syntax, we use what is called a metalanguage.
NOTE: I might use SyntaxTemplates or SyntaxDiagrams to teach the C++ syntax, but I will probably not use them much because in my opinion they aren't needed all that much.

Syntax Templates

Syntax templates are one type of metalanguage that uses graphic conventions with symbols and words to depict the syntax of a programming language. We use boldface characters to depict a literal word/C++ phrase, and non-bold characters/words can be defined using their own SyntaxTemplate.

Syntax Diagrams

Syntax diagrams are basically diagrams that show the syntax of a programming language. There are a variety of types of syntax diagrams that we can use. From the way that I see these terms used, SyntaxDiagram is a broader term encompassing many metalanguages, while SyntaxTemplates are a specific type of SyntaxDiagrams.

Basic Syntax

There are 4 basic structures for expressing actions in a programming language: sequence, selection, loop, and subprogram.

  • Sequence → A sequence is just what you think it is - a group of statements executed in consecutive order.
  • Selection → Selection is when the true logic comes into play. In a selection statement, the computer looks at some boolean expression, and depending on the result (true/false) it will do execute one group or statements or another group of statements. We often call such a statement a conditional statement
  • Loop → A loop statement is a statement that the computer repeats, possibly tweaking along the way, until a certain condition is meant. Once complete, the computer moves on to following group of statements (in the overall sequence of things).
  • Subprogram → Like in all tasks, it is often useful to break up large tasks or programs into smaller tasks or subprograms. A subprogram is a block of code that is executed in sequence. Once the subprogram is completed, execution is returned to the rest of the normal sequence. Note that in most programming languages subprograms are called functions

If we consider a simple program, it will be a sequence, a list of consecutive statements. Speaking of which, it would be useful to define what exactly a "statement" is in a programming language. In machine-language, each statement tells the computer to do exactly one thing, such as adding two quantities, or storing that result into a memory location. As our programming languages get higher, each statement is actually doing a handful of actions. Whenever we write const int sum = A + B; in C++, the computer is doing several actions, such as adding A and B, storing that result in a specified memory location, and storing the name ('sum') in a special memory location as well.

So that is the basics of syntax and semantics. We focused mainly on syntax and the overall layout of a program, as well as tools such as SyntaxTemplates, because semantics is more language-specific. As we begin our study of C++ specifically, we will begin to look more into the semantics, as well as the specific syntax using metalanguages to help us.

Introduction to Algorithms.

As we saw when looking at the syntax of programming languages, a computer program is essentially just a sequence of computer statements. Of course, when we begin to write more complicated programs we will use more complicated structures within that overall structure, such as subrpograms and conditionals, but when it comes down to it we are always dealing with a sequence of steps to accomplish our goal.

Whenever we write computer programs, we are following a specific algorithm. Computer programs are basically algorithms applied to a computer, as in their definitions below.

  • Algorithm → Algorithms are a step-by-step process that is designed to accomplish a goal in a finite amount of time.
  • Computer Program → A sequence of instructions written to be executed by a computer.

We create algorithms all the type for all sorts of projects--anytime we need to get a complicated task done, we need to break up the task in a set of steps designed to be done in order (an algorithm). In the problem-solving phase of programming, we plan out how we want to make our program to look. Although it can be tempting to simply jump into programming, it will definitely save you time if you write down a methodical process before trying to code. Also, note how an algorithm is desinged to complete some objective, not just work away at a task forever.

Many times programming teachers will intoduce programming by asking their students to create an algorithm to do some simple task, like tying a shoe or making a PBJ. The reason they do this is because as humans learn we are constantly combining simple tasks to form more complicated tasks, forming aggregate tasks. Then, once we've learned how to do these complicated tasks, we often make assumptions on how to do that task because we are so used to do it in one swift action. For example, once you learn how to type, if you think "type 'the'", then your fingers simply type the corresponding characters, without you needing to look where the correct keys are, and how to move your fingers there. You have learned from past experience (often termed 'mucsle memory') how to do these tasks.

So in programming, we will need to stop thinking in terms of "aggregate algorithms", and really think about how complete a task, one step at a time.

History of Programming & C++.

In this section I just want to go over a brief history of programming and then how C++ evolved from the previous programming languages.

The birth of high-level languages

John Backus developed the first high-level programming which was called Fortran, which stands for FORmula TRANslator, its full title being The IBM Formual Translating System. Fortran introduced many new ideas such as arithmetic operations, comments, IF statements, and the DO loop (analogous to the FOR loop used in programming languages to day). Although it seems so easy now, the problem to convert high-level expressions (A = (B + 2)/4) into machine code was a huge task.
NOTE: International Business Machines Corporation (IBM) is known as being one of the best computer-related companies. One of their achievements is creating Fortran, as discussed here.

Still, Fortran was designed to work on IBM computers, and it still wasn't all that compatible with all computers in general. To fix this, the Committee on Data Systems Languages was formed, advised by Grace Hopper, to help develop a "common" programming language that could be used across multiple computers. In response, the Common Business-Oriented Language (COBOL) was formed. Although Grace Hopper did contribute to the creation of COBOL, the real credit goes to Mary Hawes who came up with the general idea. But with the creation of COBOL, we could finally write software once and use it on any computer (assuming the computer had a COBOL compiler), a notion called "Write Once, Run Anywhere."

  • Originally (even on these high-level progrmaming languages), computers were programmed on literal punch cards.

After the creation of of the giants FORTRAN and COBOL, other programming languages were developed to follow a more specific, precise syntax. Such languages included Algol 60 and its successor, Pascal. One of the reason Pascal was created was because Algol 60 was primarily built for mathematical operations.

Origins of C++

Many of the worlds most popular languages (C, C++, C#, Java, etc) all have roots from C. C was so-named because it came from a programming language called B, which was derived from the Basic Combined Programming Language (BCPL). Dennis Ritchie was the author of the C programming language, which he created while he was at AT&T Bell Labs. C was initially created to run on the UNIX operating system (not the exact same thing as LINUX), although it has since been integrated into all the major operating systems.

Now, in 1985, Bjarne Stroustrup (from Bell Labs as well) created the C++ language. The "++" is an increment operator found in many high-level languages, and thus C++ is basically an upgraded version of C. As Stroustrup wanted a flexible/dynamic language like C, but also with additional features that encompassed Object-Oriented-Programming features, he created C with Classes.

  • NOTE: C with Classes was renamed C++ in 1983.

One of the most notable features in C++ is its use of Object-Oriented-Programming (OOP). Although Object-Oriented-Design (OOD) is not required in programming, many programmers use it for all types of projects. There is actually some debate on whether OOD or vanilla functions are better, but each of them certainly have their places.
Anyway, that is the basic history of C++ and programming languages in general. One thing to keep in mind is that as C++ evolved, the changes were not always universal (so there was not always standard compatibility to go around). To fix this, the International Standards Organization (ISO) and the American National Standards Institute (ANSI) began creating a C++ standard. Finally, in 1998, the ISO/ANSI language standard for C++ was finalized, now simply called standard C++. So that is the C++ standardized, high-level, programming language that we will be coding in.

Ethics of Programming.

Just like in everything we do, we should strive to maintain a high moral integrity. For programming, this primarily means respecting others' ideas and not trying to "steal" them. Just like writers shouldn't plagiarize others thoughts or ideas, we shouldn't steal others ideas and claim credit as our own. Provided without further information, you shouldn't be taking code from a programmer without asking permission. Additionally, it is not wise to go about sharing specialized code with others unless it isn't a big deal if someone steals it.

Now, many countries have copyright laws. As it is the duty of the government to protect the librerty and property of its people, the contract here is that the government will protect the property of the people, and in return, the people will strive to come up with more creative ideas. Now, over time various copyright licenses have been created. Anything that is in the public domain is open to anyone to use for any use, including commercial, without any need of credit. There are several copyright licenses and/or copyright ideas that you should be familiar with.

  • Creative Commons (CC) → Although there are a variety of types of CC licenses, a CC license generally lets you use the content as long as you give credit. A variety of other restrictions might be imposed like not allowing commercial use, or prohibiting you from creating derivations off it. For more detail, check out the CreativeCommons website.
  • MIT License → The MIT License is a permissive license that lends it self to making open source software. It originated from the Massachusetts Institute of Technology (MIT), where it got its name from. The main restriction imposed on MIT is that they must include the same MIT license with their project.
  • Fair Use → The idea of "fair use" is that creative works can be used in certain exceptional cases, that would normally be restricted under the copyright law. Fair use is designed to balance the intrests of the authors of creative works with the public interest. Various factors determine whether something can be used under the fair use doctrine: the purpose (commercial v. nonprofit), the nature (fiction v. fictional: facts and ideas are not protected by copyright, only their particular expression is), how much material is used, and the resulting effect of using it.
  • Public Domain → Anything that is in the public domain can be used by anyone. Eventually, everything will enter into the public domain, but the time until copyrighted materials enter into the public domain is typically getting longer.
  • No License → If no license is given for a creative work, then it is under exclusive copyright by defualt. This means that nobody else can copy, distribute, or modify your work (legally speaking). This means that must ask the author for permission before taking code from an unlicensed project.

Many times the copyright license for a coding project (or otherwise) will be listed in a .readme file.
If you are coding a project as a job, say if you are software engineer designing code for a company, then you will most likely want to restrict your code from being used by others, especially for commercial uses.     // so we see that we type the structure of a declaration is like so: 
    // DATATYPE IDENTIFIER;
    
    // let's see some other example
    const int SOME_CONSTANT = 3.14;
    
    /** in this example, we added the keyword 
    "const" to denotate that we are dealing with 
    a constant and not a variable **/
    
    
    return (0);
}

Let's review the 3 parts of a declaration. (1.) The data type can be a numerical data type like int or float or a string data type like char or string.
(2.) The identifier is named in a variety of conventions, depending on what type of identifier we are dealing with. Regardless of which convention you are following, you must start the identifier with a letter or underscore (_), and you can only use letters, underscores, or numbers throughout the name.
(3.) Finally, we end our declaration statement with a semicolon. In C++, the compiler will break if we don't put a semicolon denotating the end of the statement (when we need to). There are some cases when we don't need to use semicolons.

IMPORTANT NOTE: Your coding editor will likely tell you that there is an error on the line of code following a line that is missing a semicolon. This is because the compiler finishes the line of code (missing the semicolon), and when it tries to read the next line it runs into an error because it hadn't properly finished the previous line of code.

That is the very basics of declaration statements and identifiers. As we look at each data type individually we will look at how to declare them in more detail.

Exit Status

NOTE: This section will probably not be fully understood until we look at functions in more detail. It is recommended that you come back here and reread this once you learn about functions.

Every program in C++ must have the function main. This contains all of the C++ code that will be run in sequence. As of right now, the only code that is not inside the main is the comments at the top of the program and the header libraries. Later we will look at definining other functions and elements outside of main, but regardless, main is the main program where all the execution of the program goes on.
Now, main is a value-returning function, a type of function we'll look at later. Main is returning data of type int, signified by the int before the identifier main in the main's declaration. This means that we must return a value at the end of main, which is exactly what we do. The compiler expects a number (an integer more specifically) to be returned, and we can use this to our advantage. At the end of the execution of main, if everything went correctly without any errors, we want to return a 0, which tells us the exit status. We could put any number, but the convention is to use 0. This way, when we look at the program execution, if we see 0 we know that everything went smoothly.
On the other hand, if something goes wrong during the course of the program, we could choose to return some other value to return. It could be 1, 2, 3, or whatever integer you choose. Make sure to stay consistent with what number you choose for each type of problem. Later, we will see examples of this in action. For now, just understand that we need to return a number at the end of main, giving the exit status.

That is the basic overview of C++ syntax and semantics! Our next step is going to be to learn about identifiers and data types in more detail!

Identifiers: Numerical Data Types

Integers v. Floats

There are 3 general types of data types that we assign to our identifiers. Numerical Data types (such as int or float), String Data Types (such as char or string), and the Boolean Data Type (bool). First let's look at numerical data types.

There are two different groups of numerical data types: integers and float-point numbers. Integers are the same in programming as they are in math: basically numbers that can be written without a fractional component. Floating-Point is the name that we give to numbers that are not integers in programming. Let's take a small sidetrack and talk about floating-point numbers and the problems they present.

Floating Point Errors

To give the basic run down, we call them floating point numbers because we use a floating decimal point in our storage of our decimal numbers, instead of a fixed point. A floating-point error occurs when the computer tries to store a fraction that results in a repeating decimal. When this happens, the computer might end up with a number like 0.30000000000000004 instead of 0.3. While this information is neat, it is not super useful on its own, so you might be interested in knowing more about how computers store numbers and how floating-point rounding errors can occur. If you are, check out this article for an explanation on floating-point numbers and this video for an explanation of floating-point rounding errors.

Defining Numerical Data Types

Let's look at examples of defining numerical data types, both integers and floating numbers. When defining them, we can use data types with small storage or larger storage. The data types with larger storage lets you create bigger or smaller numbers. Basically, they have more binary digits allocated to store the data in question.

int main()
{
    // a basic integer
    int someInteger;
    someInteger = 25;
    
    // a basic floating-point number
    float someFloat;
    someFloat = 25.0;
    
    // we can also use larger versions of float
    double someLargerFloat;
    someLargerFloat = 11235123.51235 * 91236189.16;
    
    return (0);
}

That wasn't too bad, was it? We can use numerical data types for variables, constants, functions, and more. The only thing we have to do now is to keep on learning!

Identifiers: String Data Types

C++ Strings

When I say string data types I am referring to the two data types that let us type characters. The "char" data type (char is short for character) lets us store a single character. The string data type is not a type of the C++ language, but a part of the C++ standard library. Let's look at examples of defining string data types (char or string).

int main()
{
    // a basic character defined with a data type of char
    char letter;
    letter = 'A'; // we use the single quotes for char data types
    
    // a basic string
    string message;
    message = ""; // null (empty) string
    message = "Hello World!"; // use double quotes for a string
    
    return (0);
}

The "string" data type (string) comes with a lot of built-in functions that we can use with it. We will look at all of the things we can do with strings later.
Note that when we type the string, we use double quotes to distinguish it from being an identifier or literal. For example, "1253" is not the same thing as 1253, and is stored differently on the computer.

Identifiers: The Boolean Data Type

Boolean Expressions

A boolean is a data type with 2 possible values: true or false. Booleans were named after George Boole who had some considerably accomplishments when it came to computer science. Of course, it isn't all that useful if all we do is set a bool variable directly to true or false, so what we usually do is use a boolean expression. Let's check out examples in code:

int main()
{
    // declare variables
    int number;
    bool numberIsLarge;
    
    // assign our integer a value
    number = 25;
    // assign our bool variable to true or false, 
    // depending on if the boolean expression (number > 100)
    // results in true or false
    numberIsLarge = (number > 100);
    
    // in this case, numberIsLarge results in false.
    
    return (0);
}

Variables vs. Constants

Variables and Constants

The diea of a variable or constant in programming is largely similar to that in real life or math. Whenever we define a variable in programming, we are storing dating (referenced by the identifier) in a memory location space. Data stored as a variable can change (vary) throughout the execution of the program. Data stored as a constant cannot change (it remains constant) in the execution of the program.

Additionally, the way that we define variables and constants are slightly different, both syntactically and conventionally.

int main()
{
    // examples of declared variables
    int sadInt = 2;
    float happyFloat = 13.0;
    double angryFloat = 7.3
    bool areYouAmazing = true;
    // The convention for naming variable identifiers
    // is to use camel casing: capitalizing
    // the beginning of every word except the first
    
    // examples of constants
    const int COOL_CONSTANT = 8;
    const float A_COOL_NUMBER = 2.71828;
    const double DOUBLY_COOL = 5.43656;
    
    // for constants, the naming convention is to use all caps
    // with words separated by underscores
    // additionally, you have to include the keyword
    // "const" at the beginning of the declaration statement
    
    return (0);
}

Using int and float together.

Introducing Mixed Mode

Let's look at a basic example where we try to assign the wrong data type to a variable.

int main()
{
    // declare vairables
    int someInt;
    float someFloat;
    
    // this is wrong!
    someFloat = 3;
    /** Above, we tried to assign an integer 3 to
    a variable whose data type was float.
    This is not possible: what the computer does, then,
    is converts the integer 3 into a float 3.0,
    and then stores that value into someFloat.
    This extra step of converting an integer into a float
    (or vice versa) is called type coercion. **/
    
    return (0);
}

Type coercion is the implicit/automatic way of converting a value from one data type to another. We can cause type coercion when we try to assign an integer to a float variable or a float to an integer variable.
In this case, it woud simply be less confusing if we avided mixing data types. In other cases, type coercion can even lead to errors, so to avoid confusion and errors, we should convert data types when necessary, as shown below.

int main()
{
    // declare vairables
    int someInt;
    float someFloat;
    
    // this is right: explicit conversion
    someFloat = float(3 * 2 - 5);
    // whenever we convert to an integer (whether implicity
    // or explicity), the computer chops off (truncates)
    // everything past the decimal point. 
    someInt = int(3.0 / 2.0 - 0.3);
    // someInt = int(1.5 - 0.3) = int(1.5 - 0.3) = int(1.2) = 1
    
    return (0);
}

In this case, we explicity converted data types, an action called type casting (type conversion). In some instances, the only difference is clarity. Whenever we deal with any type of data conversion, we must always be careful to avoid errors, whether converting explicitly or implicitly.

More on Data Types

Categorizing Data Types

Earlier we classified data types in 3 main groups based upon whether we were dealing with numbers, strings, or booleans. Now we will classify them based upon their more technical aspects. These categorizations have more to do with how they are stored in their binary form than to how they are used.

  • Integral Types → The integral types are short, int, long, and char. All of these integers take up exactly one byte of information (at least 8 bist). This distinctino makes computer integers slightly different from mathematical integers. Additionally, we can add the keyword unsigned to any of these integral types to store only positive values.
  • Floating → Floating-point types include the float, double, and long double. The only differences here are the number of bits used to store the float-point number.
  • Address → The address data types are pointer, and reference: they are used with conjunction of other variables, dealing with how and where the variables are stored or referenced. We will not use these data types for a while.
  • Structured → The structured data types include array, struct, union, and class. These data types don't just create a single piece of data to be stored, but instead are a structured set of organized data.

The Floating and Integral Data types are often categorized together as simple data types.

Storage Limits

By now we have referenced several times that we can use different data types depending on how much storage we will need to store data. For example, if we want to store an numerical integer, we can use short, int, or long, depending on how many bits we will need to store the integer. For really large or small numbers we will need more bits. If the program tries to store a data value that exceeds the allocated memory space for that identifier then the result is integer overflow. This is an error that occurs anytime the computer tries to store a number that it doesn't have enough space to store. This can cause a variety of errors in the resulting number.
